## Key Architecture Changes to Note

### New File Structure (Phase 4)
```
src/
  components/
    chat/              # 8 files (Wave 1) — Avatar, ChannelItem, MessageBubble, etc.
    encounters/        # 5 shared + 16 section components (Wave 2)
      sections/        # VitalsSection, MARSection, PhotosSection, etc.
    incidents/
      cards/           # 14 card components (Wave 3)
      SortableCard.tsx
    shared/            # EditField, LocationEditField, StatCard
  hooks/
    useEncounterData.ts  # 294 lines — all encounter data fetching
    useIncidentData.ts   # 596 lines — all incident data fetching
  pages/
    encounters/
      pcr-steps/       # 6 step components + types.ts (Wave 2)
  constants/
    nemsis.ts          # 33 NEMSIS option arrays (Wave 1)
  utils/
    chatHelpers.ts     # Time formatters, normalizers, channel icons
    incidentFormatters.ts # calcDays, formatDeployDate, fmtCurrency, etc.
```

### Role System Change
- Removed `MD/DO` composite role entirely (26 files)
- Doctors are now `MD` or `DO` — never the combined form
- Role type: `'MD' | 'DO' | 'NP' | 'PA' | 'RN' | 'Paramedic' | 'EMT' | 'Tech'`

### Chat Privacy Model
- `is_owner` flag on employees table (white-label, no hardcoded emails)
- Owner sees all DMs silently — no membership row, invisible to participants
- DM names for owner show both parties: "Alice ↔ Bob"
- Other admins see only their own DMs + company/incident/unit channels
- Admin auto-join in ensure-channels skips DM channels entirely

### Incident Dashboard Filters
- Unit filter: pills on desktop, dropdown on mobile
- Date filter: 7d/30d/90d pills on desktop, dropdown on mobile
- Both filter encounters, MAR, supply runs, comp claims
- Card item lists capped at 5 with expand toggle
